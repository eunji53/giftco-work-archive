# giftco-tool

상품명에서 브랜드/원산지/단위/규격/특징/상품명/모델명/선택사항/불필요 등 카테고리를 추출·검수하기 위한 tkinter GUI 툴 모음입니다.

## 실행 방법 (공통)

```bash
pip install pandas openpyxl
python <파일경로>
```

실행하면 파일 선택 창이 뜹니다. 대상 엑셀(.xlsx)/CSV 파일을 선택하면 바로 작업을 시작할 수 있습니다.

## 폴더 구조 및 툴 목적별 선택 가이드

목적에 따라 아래 3개 폴더 중 하나를 사용하세요.

```
src/
├── category_label_review/       # ① 상품 하나씩 순서대로 카테고리 라벨 검수
├── product_name_tokenizing/     # ② 상품명 원본을 단어로 쪼개서 처음부터 라벨링
└── ml_category_review/          # ③ 이미 분류된(ML 등) 결과를 검수·수정
```

---

### ① `category_label_review/` — 상품 단위 순회 검수

**언제 쓰나요**: 이미 "추출단어" + 카테고리 컬럼이 있는 파일을, 상품 하나씩 순서대로 넘겨보면서 카테고리를 확인/보정하고 싶을 때.

**필수 입력 컬럼**: `상품번호`, `실제 상품명`, `추출단어`, 카테고리 컬럼들(`브랜드`,`원산지`,`단위`,`규격`,`특징`,`상품명`,`모델명`,`선택사항`,`불필요`)

| 파일 | 특징 |
|---|---|
| `category_label_reviewer_v1.py` | 기본형. 체크박스로 행을 선택해 `[단어][단어]` 형식으로 분리, 단어 입력으로 통합 |
| `category_label_reviewer_v2.py` | v1 + 상품번호로 검색 이동, 이전 위치로 돌아가기 |
| **`category_label_reviewer_v3.py`** ⭐ | v2 + 저장 시 카테고리를 하나도 지정하지 않은 행에 자동으로 "확인필요" 표시 (검수 누락 방지) |

**사용 순서**: 파일 실행 → 파일 선택 → 상품별로 단어에 라디오버튼으로 카테고리 지정 → 필요시 체크박스로 행 선택 후 분리/통합 → "저장"

**어떤 버전을 써야 하나요**: 특별한 이유가 없다면 **v3을 사용하세요.** v1/v2는 v3에 없는 기능이 없으므로 실사용 중단을 권장합니다.

---

### ② `product_name_tokenizing/` — 상품명 최초 분해 + 라벨링

**언제 쓰나요**: "실제 상품명"만 있는 원본 데이터에서, 단어 추출부터 새로 시작해야 할 때 (아직 추출단어가 없는 최초 작업).

**필수 입력 컬럼**: `상품번호`, `실제 상품명` (그 외 컬럼은 없어도 됨)

| 파일 | 특징 |
|---|---|
| `product_name_tokenizer_labeler_v1.py` | 상품명을 자체 토크나이저로 분해(괄호 처리 포함) 후 카테고리 지정. 분리는 `[단어]` 형식 |
| **`product_name_tokenizer_labeler_v2.py`** ⭐ | v1 + 단어 라벨 클릭 시 수정입력창에 자동으로 채워지는 편의 기능 |

**사용 순서**: 파일 실행 → 원본 상품 파일 선택 → 자동 분해된 단어들에 카테고리 지정 → "분리"/"통합"으로 보정 → "저장"
**출력**: 작업 완료분은 `save_data.xlsx`, 아직 카테고리를 하나도 안 지정한 상품은 `next_data.xlsx`로 자동 분리 저장됩니다(실행한 폴더 안에 생성).

**어떤 버전을 써야 하나요**: **v2를 사용하세요.**

---

### ③ `ml_category_review/` — 기존 분류 결과 검수 (ML 결과 등)

**언제 쓰나요**: 추출단어 + 카테고리가 이미 채워진(예: 머신러닝으로 1차 분류된) 결과가 맞는지 확인하고 고칠 때. 이 폴더 안에서도 두 가지 방식이 있습니다.

**필수 입력 컬럼**: `상품번호`, `실제 상품명`, `추출단어`, 카테고리 컬럼들

#### 방식 A. 상품 단위로 순회하며 검수

| 파일 | 특징 |
|---|---|
| `ml_category_review_tool_v1.py` | 상품을 하나씩 넘기며 추가/수정/분리/통합 가능. 저장(체크박스로 "작업완료" 표시) 시 다음 실행에서 이어서 작업 가능(세션 재개). 1회 실행취소 지원. 분리 시 `단어\단어` 형식 사용 |

**출력**: `save_data_MMDD.csv`(작업 완료분), `next_data_MMDD.csv`(남은 작업분)이 실행 날짜 기준으로 생성됩니다.

#### 방식 B. 특정 단어를 검색해서 한번에 검수 (추천)

같은 단어가 여러 상품에 걸쳐 잘못 분류된 경우, 상품을 하나씩 도는 것보다 **단어를 검색해서 한 번에 고치는 것이 훨씬 빠릅니다.**

| 파일 | 특징 |
|---|---|
| `word_search_category_reviewer_v1.py` | 추출단어를 검색(완전일치/포함검색)하면 그 단어가 등장한 모든 상품이 한번에 나열됨. 카테고리별 통계 제공. 여러 파일 동시 로드 가능 |
| `word_search_category_reviewer_v2.py` | v1 + 정규식 검색 (검색어에 `$d`를 넣으면 숫자로 치환해서 검색, 예: `SIZE$d` → SIZE1, SIZE22 등 매칭) |
| **`word_search_category_reviewer_v3.py`** ⭐ | v2 + 상품 상세 패널(개별 상품에서 추가/수정/분리/통합/제거) + 여러 상품을 체크해서 카테고리를 한번에 일괄 적용하는 패널 + 전체 작업에 대한 실행취소 |

**사용 순서**: 파일 실행 → 파일(복수 선택 가능) 선택 → 검색창에 확인하고 싶은 단어 입력 → 검색 결과로 나온 상품들의 카테고리를 콤보박스로 수정(또는 체크 후 일괄 적용) → "저장" (저장 파일명은 실행 시 직접 지정)

**어떤 버전을 써야 하나요**: **v3을 사용하세요.** v1/v2는 저장 시 "직전에 검색한 단어의 수정 내용만 반영되고, 그 전에 다른 단어를 고친 내용은 저장 파일에서 유실되는" 문제가 있습니다(v3에서 해결됨). v1/v2로 검수 작업을 하고 있었다면, 한 세션에서 단어 하나를 고칠 때마다 매번 저장했는지 확인해보시길 권장합니다.

**방식 A vs B, 뭘 써야 하나요**: 상품 자체를 처음부터 끝까지 훑어야 하면 A(`ml_category_review_tool_v1.py`), 특정 단어의 분류만 빠르게 고치고 싶으면 B(`word_search_category_reviewer_v3.py`)를 사용하세요.

## 공통 주의사항

- 카테고리 값: `브랜드`, `원산지`, `단위`, `규격`, `특징`, `상품명`, `모델명`, `선택사항`, `불필요` (모든 툴 공통)
- 저장 파일은 실행한 파일과 같은 폴더(또는 저장 대화상자로 지정한 위치)에 생성됩니다. 원본 파일은 덮어쓰지 않습니다.
- 실제 상품 데이터(엑셀/CSV)는 이 저장소에 포함되어 있지 않습니다. 각자 보유한 파일로 실행하세요.
