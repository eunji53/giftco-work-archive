# 작업기록 (2026-07-09)

고려기프트 발주서 PDF 수집/파싱 관련 코드를 `파일 분류/고려기프트/`에서 골라 output을 제거하고, `goryeogift/` 폴더에 영문 파일명 + 순서 번호로 정리했다. 원본은 `파일 분류/고려기프트/`에 그대로 보존.

## 파일별 설명

- **`00_crawl_orders.ipynb`**
  고려기프트 파트너 사이트(adpanchok.co.kr)에 셀레니움으로 접속. 수동 로그인 후 발주 목록 페이지까지 이동하면(`input()`으로 대기) 이후 자동으로 "승인" 클릭 → 발주서 PDF 다운로드를 반복. 이미 받은 파일(`고려기프트_XXXXX.pdf`)은 건너뛰어서 재실행 시 이어받기 가능.

- **`01_diagnose_pdf_text.ipynb`**
  PyPDF2로 특정 PDF 1개를 원문 추출해서, 상품명에 전화번호·발송처 정보가 섞여 들어가는 문제의 원인을 진단하는 디버깅용 노트북. 문제가 생긴 파일명을 직접 지정해서 원문을 눈으로 확인하는 용도.

- **`02_extract_and_convert.py`** (메인 파이프라인)
  `PDF_DIR`의 PDF를 전부 순회하며 페이지별 텍스트를 추출하고, 정규식으로 발주일/납기일/거래처/발송처(상호·성명·전화·주소)/품목(품목코드·상품명·수량·단가·공급가액·부가세·합계)을 파싱. 회사 템플릿 엑셀(`TEMPLATE_PATH`)의 컬럼 구성에 맞춰 변환한 뒤 `OUTPUT_DESKTOP` 경로에 저장.
  ⚠️ 아직 로컬 절대경로(`C:\Users\USER\...`)와 거래처 실명("명성")이 하드코딩되어 있음 — 추후 정리 예정.

- **`03_analyze_multi_item_orders.ipynb`**
  변환된 엑셀에서 파일 하나에 상품이 여러 개 들어있는 경우의 구조를 분석. 품목코드가 `X`로 시작하지 않으면 메인 상품 행, `X`로 시작(`XP000`, `XC000`, `XG000`, `XD000` 등)하면 직전 메인 상품에 딸린 옵션 행으로 구분.

- **`04_flatten_to_one_row_per_item.ipynb`**
  메인 상품 1개 = 1행으로 정리. 옵션 행(택배비 등 하단 공통 옵션 포함)은 직전 메인 상품의 `부대비용`으로 합산해서 `총비용` 컬럼을 추가.

- **`05_detect_broken_item_names.ipynb`**
  PDF 텍스트 추출 과정에서 잘리거나 깨진 상품명을 찾아내는 QA 노트북. 수량·단가·공급가액·부가세·합계가 모두 0이 아닌 행만 필터링해서 점검.

- **`06_compare_versions.ipynb`**
  구버전 결과(`고려기프트_발주서_정리.xlsx`)와 신버전 결과(`발주서/고려기프트_발주서_2.xlsx`)를 파일명+상품명 키로 비교.
